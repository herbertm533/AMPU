# Draft-Class Authoring Playbook

**Purpose.** This file is the persistent memory for building AMPU draft classes
from historical sources (Wikipedia election pages, etc.). When the user sends a
new page, **read this file first**, then follow the workflow below so every
addition is consistent with what came before. It captures the *judgment calls*
and conventions established interactively — the pipeline mechanics live in
`CLAUDE.md` ("Draft dataset pipeline").

> One-line summary: the bundled ~18.5k politician dataset is generated by
> `scripts/legislatorsToDataset.mjs`. New historical figures who **never served
> in the U.S. Congress** are hand-added to `ERA_ROWS` in `scripts/seedDataset.mjs`;
> people who *did* serve are already in the data and should not be re-added.

---

## Environment facts (don't relearn these)
- **Wikipedia is network-blocked here** (WebFetch → 403; Bash host allowlist
  rejects `en.wikipedia.org` and the API). So the user **uploads PDFs** of the
  pages; read them with the Read tool. Do not try to fetch Wikipedia.
- **`raw.githubusercontent.com` IS reachable.** That's how the base datasets are
  pulled (`scripts/fetchLegislators.sh`): `unitedstates/congress-legislators`
  (everyone who served, real birthdays) + MEDSL house/senate/president CSVs
  (failed candidates 1976+). Source files land in `.legis/` (gitignored).

## The three curation tiers (in `scripts/seedDataset.mjs`)
1. **`CURATED_ROWS`** (built from `ROWS`) — marquee in-game figures (the 1772 +
   1856 scenario rosters and key founders). These **OVERRIDE** any same-named
   person from the bulk dataset, so they keep hand-tuned stats. Edit a `ROWS`
   entry to *refine* an existing marquee figure.
2. **`ERA_FIGURES`** (built from `ERA_ROWS`) — historical notables who **never
   served in the U.S. Congress** (so are absent from congress-legislators):
   Continental Congress members, generals, judges, serious election losers.
   Merged **ADDITIVELY** with **era-aware disambiguation** (below). This is where
   new figures from election pages go.
3. **MEDSL failed candidates** (auto-derived, 1976+) — added additively with
   sub-floor stats. Not hand-authored.

Merge precedence (in `legislatorsToDataset.mjs`): served members (real data)
< `CURATED_ROWS` (override) ; `ERA_FIGURES` and failed candidates only added
when no same-name person already occupies that era.

## The bar for inclusion (who to add to `ERA_ROWS`)
ADD a figure only if **both**:
- They are **notable** — a recognizable historical actor (signer, Continental
  Congress delegate, general, governor, chief justice, a serious challenger who
  lost a real race), *not* a 6-vote fringe/scattering name.
- They **never served in the U.S. Congress** (House or Senate). If they served,
  they're already in congress-legislators — skip them. (Quick mental check: a
  winner of a seat almost always served; focus on the *losers* and on people
  whose fame is pre-/non-congressional.)

When unsure, lean **out**. A smaller, accurate set beats noise.

## The balance rule (non-negotiable)
Added era figures and failed candidates must be **unlikely to win elections**
(PV drives `calcStateVote`). So give them **sub-floor electoral stats**:
- `legislative` ≤ **1** (the elected baseline floors at 2 — this is the key lever)
- `command` low (0–2; 0 for most, 1–2 only for genuine national stature)
- **no positive electoral traits** like `Orator`/`Charismatic`/`Manipulative`
  unless truly defining; flavor traits (`Military`, `Egghead`, `Naval`) are fine.
Their *non-electoral* skills can reflect their real role (a general gets high
`military`; a judge high `judicial`) — that's color, not an election threat.

## Era-aware disambiguation (the "John Sullivan" rule)
Names repeat across centuries. The merge treats a name as the **same person /
skip** only if an existing same-named entry was born within
`ERA_SAME_PERSON_WINDOW = 25` years. Otherwise it's a **different** historical
person and is added under a distinct key (`name#birthYear`). This is why
Gen. John Sullivan (b. 1740) coexists with a Congressman John Sullivan (b. 1868).
Practical implication: **always set an accurate `birthYear`** — it's what keeps
people distinct. You don't need to hand-handle collisions; the merge does it.

## `ERA_ROWS` row format
```js
// [first, last, state, ideology, birthYear, [adm,leg,jud,mil,gov,bck], command, [traits], party]
['John','Sullivan','nh','Moderate',1740,[1,1,1,4,2,1],2,['Military','Leadership'],'Pro-Administration'],
```
Field-by-field:
- **state** — lowercase id. Founding colonies/states: `nh ma ri ct ny nj pa de md va nc sc ga`.
- **ideology** — 7-point scale (see vocab below). Map from faction:
  - Pro-Administration / Federalist → usually `Conservative` (or `Moderate`;
    `Traditionalist` for high-church/aristocratic).
  - Anti-Administration / Anti-Federalist / Democratic-Republican → usually
    `Liberal` (or `Progressive`; `Traditionalist`/`RW Populist` for
    states'-rights agrarians like Patrick Henry).
- **birthYear** — real, accurate (drives draftYear *and* disambiguation).
- **skills `[adm,leg,jud,mil,gov,bck]`** — integers 0–5. Pick the 1–2 that match
  their role and set 2–4; keep `legislative` ≤ 1; leave the rest at 0–1:
  - General/officer → `military` 3–4 + trait `Military`
  - Lawyer/judge/justice → `judicial` 2–4 + trait `Egghead`
  - Executive (governor, Pres. of Continental Congress) → `governing` 2–3, `command` 1–2 + `Leadership`
  - Financier/merchant → `admin` 2–3 + `Economics`/`Business`
  - Naval officer → `military` 2–3 + `Naval`
- **command** — 0–2 (see balance rule).
- **traits** — array from the canonical vocab (below). 0–2 is typical.
- **party** — the historical label as a string (`Pro-Administration`,
  `Anti-Administration`, `Federalist`, `Anti-Federalist`, …). **Review-only**:
  the importer ignores it; it's there for the CSV reviewer.

## Workflow for a new election page
1. **Read** the uploaded PDF.
2. **Extract candidates.** For each district/state, list the people. Separate
   *winners who served* (skip — already in data) from *losers / never-served
   notables* (candidates for `ERA_ROWS`).
3. **Apply the bar.** Keep only notable, never-served figures.
4. **Author rows.** For each, fill the `ERA_ROWS` format using the heuristics
   above. Accurate `birthYear`; sub-floor electoral stats; role-matched skills;
   faction→ideology; flavor traits.
5. **Marquee refinements (optional).** If the page reveals something about an
   existing `ROWS` figure (e.g. "first Speaker", "beat Sam Adams"), refine that
   `ROWS` entry — these override, so it's safe and improves an in-game figure.
6. **Regenerate & build:**
   ```bash
   node scripts/legislatorsToDataset.mjs && npm run build
   ```
   (`legislatorsToDataset.mjs` imports `CURATED_ROWS` + `ERA_FIGURES` from
   `seedDataset.mjs` and rewrites `public/standard-draft-classes.json`,
   `politicians-dataset.csv`, and `src/data/defaultDraftClasses.ts`. You do **not**
   need to run `seedDataset.mjs` separately — that only writes the curated CSV.)
   `.legis/` must be present; if missing, run `bash scripts/fetchLegislators.sh` first.
7. **Sanity-check** the console line ("added N era figures (M skipped …)") and
   spot-check a few new names in `politicians-dataset.csv` (draftYear, sub-floor
   `legislative`, era disambiguation). Confirm `npm run build` passes.
8. **Commit** with a clear message; **push only with the user's approval**
   (per CLAUDE.md — pushing is a human checkpoint). Then update the progress
   tracker at the bottom of THIS file.

## Canonical vocab
**Ideologies (7):** `LW Populist, Progressive, Liberal, Moderate, Conservative, Traditionalist, RW Populist`

**Positive traits:** `Charismatic, Integrity, Efficient, Orator, Debater, Propagandist, Crisis Manager, Kingmaker, Numberfudger, Harmonious, Manipulative, Celebrity, Magician, Naval, Education, Economics, Business, Agriculture, Environment, Media, Nationalist, Globalist, Reformist, Military, Egghead, Leadership`

**Negative traits:** `Incompetent, Passive, Unlikable, Puritan, Domestic Apathy, Flip-Flopper, Corrupt, Scandalous, Frail, Controversial, Obscure, Traitor`

(Traits must match exactly — see `ALL_TRAITS` in `src/data/draftImport.ts`. The
importer drops unrecognized traits.)

**draftYear math:** nearest multiple of 4 to `birthYear + 25`, clamped to ≥ 1772.
Computed automatically — you only supply `birthYear`.
