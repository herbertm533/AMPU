### Constitutional Convention (~1788) — corroborates #159

- Proposal order by legis ability (rolled tiebreak). Per-article **propose → vote**
  (1 state = 1 vote). Articles: **1** Congress (bicameral vs unicameral), **2**
  Presidency (4-yr elected; default if only option), **3** SCOTUS, **4** Amendment
  threshold (2/3 vs majority), **5** Pres eligibility (foreign-born resident vs any),
  **6** slave count (3/5 vs full), **7** women suffrage. (POST 2218-2230)
- **Required articles:** Art 1, Art 2, Art 3, **Slave-State Compromise**, and an
  **Amendment process** MUST pass; one option per article. (POST 2232)
- **★ NEW rule added live: "After 3 inconclusive votes, CPU votes for the
  HISTORICAL bills to keep the game moving."** Strongly corroborates #159's
  "CPU keeps things historical" bias + the 3-strikes pattern. (POST 2232)
- Pres **can't propose** (rule added live). Optional planks bound to a proposing
  pol's interest (e.g. Civil Rights). (POST 2219/2232)
- Convention President = celebrity + 1 command + military-leader (auto), else vote
  among ≥1-command delegates, else lowest-score player picks. **"Father of the
  Constitution"** = the Egghead drafter (here George Wythe). (POST 2190-2192)
- **Outcome (near-historical this run):** bicameral, 4-yr Pres, SCOTUS, 2/3
  amendments, foreign-born-resident eligible, 3/5 slave count, **no women suffrage,
  slavery not banned.**

## C. CPU / solo-mode & GM-workload signal (corroborates #114/E9/K5, DH-36)

This is a **multiplayer GM run**, so it's a useful **contrast** to the solo runs:

- **CPU runs only 1 faction per party at start**, then **expands to fill every
  vacated human faction** — by federalism the GM is **manually simming up to 5 CPU
  factions per party**, repeatedly described as painful ("Federalist/Anti-Fed
  papers freaking complicated when controlling 5 factions"). The CPU-AI the app
  must own is exactly this per-faction fallback. (POST 2146)
- **★ CPU-governor meta-math (Ted):** with all-CPU governors, 50 govs each doing
  ~3 actions at ~20% success only move a crisis meter **~1-2 spaces** — "much less
  with fewer states or multiple simultaneous crises." Human-controlled governors
  change the math. → **CPU is mechanically weak at fixing meters; a solo human
  carries the load.** Bears on #114 (solo is the target mode) + CPU-AI tuning. (POST 2671)
- **GM workload concentrated** on: simming 5 CPU factions/party, processing
  per-package multi-chamber votes with sway, the Constitutional-Convention
  per-article machine, cabinet lingering rolls, and the federalism
  decision/diplomacy spine. **3-GM relay** (vcczar → Ted → Rezi) over 14 months
  with persistent burnout-adjacent delays — reconfirms **DH-36** (GM-workload is the
  real constraint) without a clean burnout-quit (it just trails off).
- **Live rule-authoring** is constant: the enthusiasm engine (#18/#51), the
  3-inconclusive-votes→CPU-historical rule, the non-VP-succession command rule, the
  Efficient-cabinet +2 rule, the faction-leader-not-on-career-track rule — all
  written/patched **mid-run**, i.e. federalism rules were **not finalized** when the
  app would need them.
