## Which GM functions Ted automated — and the priority order

Ted states his own ordering (POST 1, restated POST 30): **Step 1 = "low-hanging fruit"** (things already known/easy) → **Step 2 = "high effort/high reward"** (the dropdown-autopopulate, which kills per-politician lookup) → **Step 3 = "the rest"** (low-value/hard, do once skilled). The actual work landed in this order:

1. **SUM-based tallies (Main tab)** — auto-count Senate/House/Governor seats per faction AND per party (POST 1, 2). First skill learned (SUM, POST 1). *Side effect:* required **standardizing job-title strings** ("Career Track Governing" → "Career Track") so COUNTIF wouldn't miscount Governors (POST 2) — a data-hygiene note (see incidental).
2. **Cross-sheet pulls** — count occurrences of "Senator"/"Rep" in titles on faction pages and surface on other tabs (POST 1). "A major hurdle." Research finding: **a dropdown cannot pull names from multiple faction sheets** (POST 12) → forced a **single Master Sheet of all active politicians** (POST 12). Excel cap: **dropdowns error >1000 names**; roster was 412 active, assumed safe if dead/retired are pruned (POST 14).
3. **Per-politician age column** — auto-computes age from the current 2-year term (POST 5, 9) "to help with deaths, retirements, and confirming everyone is the right age to run." (Eligibility data the engine needs.)
4. **THE BIG ONE — dropdown→autopopulate** (Step 2): type a few letters of a politician's name, select, and the sheet auto-pulls that politician's **faction name + election/legislation-relevant traits** (POST 13 found the method, POST 16 working). Rolled out across **Executive** (POST 20: traits), **GovStates/Governors** (POST 21: faction, gov-ability level, Efficient, Incompetent, Leadership, Iron Fist, Pliable, Puritan), **Military** (POST 22: faction, Military ability, Military Leader), **Career Track** (POST 4: per-faction statesman dropdowns), **Kingmaker** (POST 26), **Senate/House** (POST 27, 52: faction, Legislative ability, Debater, Disharmonious, Easily Overwhelmed).
5. **Election scoring — the headline** (POST 37–55). Built per-office election tabs that auto-compute nearly all inputs:
   - Governor elections **fully done** (POST 53): **41 total variables; players input 2** (candidate name + prior gov terms), **Ted manually inputs 12** (mostly die rolls + multivariable check results he eyeballs), **27 fully automated** "all on a single page."
   - **Kingmaker-per-state count** via two-cell match (State × Kingmaker) (POST 37).
   - **Eligibility / ineligible-candidate flags** auto-raised (POST 44). **Term-limits = the wall Ted couldn't clear** (POST 44, 48): he can pull *whether* limits exist + current title (incumbency) but **failed to count prior 2-year gov terms from the History tab** → term-limit enforcement left manual.
   - **VLOOKUP + COUNTIF combo** to detect a candidate holding **both Celebrity AND Military Leader → +1 in elections** (POST 50) — explicitly a *two-trait-on-same-candidate* combo bonus (see incidental gap).
   - **Senate, House elections done** by copy/paste off the Governor template (POST 54, 55; ~95% reuse, POST 50). **Presidential elections deferred** (POST 56, 62 — still to-do at thread end).
6. **Legislative outcome scoring** (POST 56–59, 62): a "Calculating scores" tab auto-computes **points awarded per faction from passed legislation based on which cards a faction holds** — single law (POST 57) then **multi-law packages** (POST 58, 59). Known gap left manual: **state-industry point gains are NOT folded in** (POST 59, 62 — the closing to-do item).
7. **Legislative vote-shifting** (POST 52): traits surfaced per-politician on one page, but **the dice rolls stay manual** ("dice rolls can be automated too, but not in the way I want them to be") → Ted keeps the human die-roll deliberately. (Contrast the app mandate: seeded RNG, never a human roll.)
8. **Cleanup / data-hygiene (Step 3 + incidental):** era-filtered Gov Actions (POST 10) and Cards-Held (POST 2 — manually hides cards illegal in the current era, *re-done each era change*, POST 2); renamed outdated cards ("Big Prison" → "Law and Order", POST 2); blanked stray "0"s in lingering/meter columns for legibility (POST 2); fixed multi-column sort corruption for dynasty/parent/spouse/special-draft/Wiki fields (POST 5, *future rows only*); deleted no-op lingering laws (POST 2); split Senate/House **leadership & committees onto their own tabs** so the member list can be filtered without breaking leadership rows (POST 31–36).

**Take-away ordering for the app:** Ted's lived priority = (a) seat tallies, (b) per-politician trait/eligibility lookup, (c) per-office election scoring, (d) legislative package scoring — and the two he *couldn't* automate (**term-limit history-counting**, **per-candidate die rolls**) map to two things the app already handles trivially (term tracking in state; seeded RNG). That inversion is the clearest argument for the app: the hardest manual GM tasks are the engine's easiest.

## Incidental mechanics revealed (data-shape)

- **Election-odds inputs are per-candidate trait flags**, surfaced per office: Governor races key on Efficient/Incompetent/Leadership/Iron Fist/Pliable/Puritan + gov-ability (POST 21); Senate/House on Debater/Disharmonious/Easily Overwhelmed + Legislative-ability (POST 27, 52); Military offices on Military ability + Military Leader (POST 22). Matches the shipped `Trait` set + `applyTraitElectionBonus` (`src/engine/phaseRunners.ts` calcStateVote → `TRAIT_ELECTION_EFFECTS`, `src/types.ts`).
- **A two-trait AND-combo election bonus exists in the design:** Celebrity **AND** Military Leader on the same candidate → **+1** (POST 50). *Verified against code:* the shipped `TraitElectionRule` (`src/types.ts:727`) supports per-trait, per-context, era-scaled, and **opponent-conditional** bumps, but has **no "candidate also-holds trait X" combo field** — POST 50's combo is not expressible in the current rule table. Small candidate gap (below).
- **Governor-election scale:** "41 variables," of which only candidate-name + prior-gov-terms are player inputs (POST 53) — a concrete sense of the per-state governor-race input surface.
- **Legislative scoring shape:** faction points from a passed law = function of (the law's meter effects) × (which **cards** the faction holds), summable across a multi-law package (POST 57–59); **state-industry effects are a separate addend, left manual** (POST 59, 62). Corroborates the legislation-scoring + lobby/industry-card model already in the gap log.
- **Roster scale (spreadsheet era):** ~412 active politicians at this point (POST 14); a single all-active "Master Sheet" is the canonical politician index (POST 12) — the spreadsheet analogue of `snap.politicians`.
