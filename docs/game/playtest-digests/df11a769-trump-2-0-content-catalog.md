## ★ The designer's inclusion bar (POST 1) — the authoring rule

Any modern-content proposal must clear ALL of:
- **Major, not minor** — "major = likely lasting impact"; "minor = no real
  impact no matter how flashy." Flash ≠ inclusion.
- **≥40% pass/act probability** — proposals that won't realistically pass/be
  acted on are excluded ("a bill to legalize enslavement of Democrats" is the
  designer's counter-example of a sub-40% non-starter).
- **No duplication** — don't re-propose what the game already has.
- Must be **"on par with what the game has for all the previous presidencies"** —
  i.e. matched in weight to the existing era-event / bill catalog.

This is a reusable **content-author acceptance gate** for the whole modern era,
not just Trump 2.0.

## Shipped-vs-designed (verified against `src/` HEAD, 2026-06-30)

**None of this thread's content ships, and the two enabling mechanics it relies
on most — predicate-gated availability and bundle bills — are either 1772-only
or entirely absent.**

- **★ NO modern-era content set ships AT ALL.** `Era` is a 4-value union
  `'independence' | 'federalism' | 'nationalism' | 'modern'` (`types.ts:1337`) —
  there is **no future/2025 era**, and only **two scenarios** exist:
  `scenario1772.ts` (`scenarioId:'1772'`) and `scenario1856.ts`
  (`scenarioId:'1856'`, `currentEra:'nationalism'`). No `scenario2025`/modern
  data file exists (`src/data/` glob = 1772 & 1856 only). Every bill, exec
  action, gov action, event, and SC case in this thread is **designed-only**.
- **★ Bundle/omnibus bills — UNBUILT (headline mechanic gap).** `Legislation`
  (`types.ts:1506-1520`) carries a **single** `effects: EraEventResponseEffect`
  blob — there is **no sub-component / rider / amendment array**, no notion of a
  container bill made of independently-authored parts. Grep of all of `src/` for
  `bundle|omnibus|component|rider|provisions|subBill` = **0 legislation hits**
  (only `constitutionalArticles.amendmentProcess` and the senate-pre-17th note).
  The per-state exemption "vote-buying rider" the Behemoth implies (POST 21) has
  no representation. So the designer's POST-20 plan ("make it a bundle bill") is
  a **net-new bill type** plus a way to compose & jointly-vote N effects.
- **★ Predicate-gated availability (#258) is wired in the 1772 GRAPH ONLY.**
  The `Predicate` tree (`types.ts:1487-1504`) and its pure interpreter
  `evalPredicate` (`eraGraph.ts:12`) + walker `selectEraGraphNode`
  (`eraGraph.ts:107-114`, `precondition ?? {all:[]}`) drive node availability —
  but `precondition` is **only authored in `eraEvents1772.ts`** (grep of
  `precondition` across `src/` = `types.ts`, `eraGraph.ts`,
  `EraEventsPage.tsx`, `EventChips.tsx`, `EraEventModal.tsx`, `eraEvents1772.ts`
  — **`eraEvents1856.ts` is NOT in the list**). 1856 events are gated by
  **hardcoded `if (year >= 1856 && year <= 1858)` blocks** with **zero**
  preconditions (`eraEvents1856.ts:6/30/55…`; grep for `requires|prereq|unlocks`
  in that file = 0). So the many **precondition-gated exec actions** this thread
  wants ("requires Reciprocal Tariff Act," "requires Jan-6 era-evo," "requires
  natural disaster," "requires high unrest + protest event," "populist
  president") have a **matching machinery that is not generalized to non-1772
  content** — that machinery would need to extend to modern era events AND to
  presidential/governor actions (which today have no predicate hook at all).
- **★ 3-primitive content model (#221) exists for EVENTS, not for actions.**
  The shipped event primitive is `EraEvent { responses: EraEventResponse[] }`
  where each `EraEventResponse.effect` is an `EraEventResponseEffect`
  (`types.ts:1448-1482`) — a **data-driven (event → response → effect)** trio
  that DOES support multiple simultaneous sub-effects (meters, enthusiasm,
  interestGroups, diplomacy, startWar) inside one effect. So a single response
  can already carry a *composite* effect — but that is NOT the same as a
  **bundle bill of separately-votable components**. Critically, **executive and
  governor actions are NOT modeled in this primitive at all** (see next two).
- **★ Executive actions — shipped as a hardcoded 4-item `pick()` stub, NOT
  data/predicate-driven content (#169/#270).** `runPhase_2_8_1_Executive`
  (`phaseRunners.ts:3632-3646`) is the ENTIRE presidential-action system: a 50%
  roll, then `pick()` of **4 fixed actions** ("streamline the Post Office,"
  "enforce federal law in territories," "pardon political prisoners," "negotiate
  a trade treaty"). It is **era-agnostic, not data-driven, has no precondition
  hook, no player choice, no content registry.** Every exec action in this
  thread (DOGE, Schedule F, Liberation Day tariffs, pardon-allies-requires-Jan6,
  send-Marines-requires-unrest, etc.) would need this stub replaced by a
  predicate-gated, era-banded, data-authored exec-action catalog. The shipped
  "pardon political prisoners" is the *only* faint overlap, and it's a flat
  partyPreference nudge, not the gated "pardon allies prosecuted by prior admin."
- **★ SCOTUS cases — type + plumbing exist, ZERO case content ships (#270).**
  `SupremeCourtCase` (`types.ts:1548-1556`) and `GameState.pendingCourtCases`
  exist, but the 1856 scenario seeds **`pendingCourtCases: []`**
  (`scenario1856.ts:175`) — no court-case content in either scenario. Skrmetti /
  the 2024-term docket would be **net-new content** on an otherwise empty
  content slot (the engine has a `runPhase_2_8_2_CourtMgmt` for justice
  retirement/replacement, `phaseRunners.ts:3648+`, but it does not *adjudicate
  authored cases*).
- **★ Governor actions — shipped as an 11-line flat-skill stub; none of the
  proposed gov actions exist (#20).** Per digest `0ff5e582`,
  `runPhase_2_5_2_Governors` is a 30% `bias`-nudge reading only
  `gov.skills.governing` + party; the National-Guard / refuse-deportations /
  governor-pacts / detain-judges / social-media-presence / rig-a-race actions
  are all unbuilt, and the GovernorsPage is read-only.
- **Government shutdown (KFJ's "probably already added," POST 4) — PARTIAL.**
  There is no dedicated shutdown *event* in `eraEvents1856.ts`, but the engine
  models budget/debt meters and a `government` meter; treat "shutdown" as
  **designed-not-built as a discrete era event** (no `shutdown` event template
  in `src/data/`).

**Net for tech-lead.** This thread is a **pure modern-era content backlog** with
**two structural prerequisites the build lacks**: (1) a **bundle-bill** type
(compose & jointly-vote N independently-authored sub-effects, incl. per-state
riders) — `Legislation` is single-effect today; and (2) **predicate-gated
availability generalized beyond the 1772 graph** — the `Predicate`/`evalPredicate`
engine is excellent but authored only for 1772 era events, while modern exec
actions (a hardcoded 4-item `pick()` stub) and gov actions (a flat-skill stub)
have **no predicate hook, no data-driven catalog, and no era band**. Plus a
**modern/Era-of-the-Future scenario + content set** to host all of it (none
ships). SCOTUS-case content is an empty slot with type+plumbing ready.
